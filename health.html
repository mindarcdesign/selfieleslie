Overall Site Health

2.2/5 ‚Äî Significant Mobile Performance Issues

Selfie Leslie faces a critical mobile performance gap that demands immediate attention. While desktop performance scores range from 46-77/100, mobile users experience severely degraded performance with scores of just 34-38/100 and LCP times reaching 10-17 seconds. The root cause is clear: LCP images are lazy-loaded on the PLP, causing a staggering 17.16-second delay before the largest content becomes visible. Combined with 20 long tasks blocking the main thread for nearly 2 seconds and a heavy third-party script stack (Yotpo, Clarity, Searchspring, Klaviyo, and 10+ others), mobile shoppers face significant friction at every stage of the purchase journey. With mobile traffic typically representing 65-75% of fashion eCommerce sessions, these issues directly impact bounce rates and conversion. The good news: the infrastructure is sound (server response times are excellent at 27-40ms), making targeted optimizations highly achievable.

Critical Issues (100% User Impact)

  • LCP Crisis on PLP: 17.16s mobile LCP due to lazy-loaded hero images
  • Severe Mobile Scores: 34-38/100 Lighthouse performance across all pages
  • Main Thread Blocked: 20 long tasks totaling 1.9-2.4s blocking time
  • Heavy Payloads: 4.5-7.1MB page weights with 266-356 requests

Moderate Risks (Third-Party Overhead)

  • Heavy Third-Party Stack: 12+ scripts adding 600+ms blocking time
  • Yotpo Impact: 203-207ms blocking time + 688KB transfer
  • Clarity Recording: 124-142ms blocking on every page load
  • Unused JavaScript: 548-712KB potential savings per page

Strengths (Foundation Solid)

  • Excellent TTFB: 27-40ms server response times
  • Good Accessibility: 76-93% scores across pages
  • Stable PDP CLS: 0.087 on mobile product pages
  • Modern Platform: Shopify with optimization potential

Mobile Revenue at Risk

With 10-17 second LCP times on mobile, users on slower networks will experience significant delays before seeing product content. Research shows each additional second of load time reduces conversions by 7-10%. At current performance levels, Selfie Leslie may be losing 40-50% of potential mobile conversions due to performance issues alone. Immediate action on LCP image preloading and third-party script deferral is critical.

Top 5 Optimization Priorities

  1. Fix PLP LCP Image: Remove loading="lazy" from first product images, add fetchpriority="high" (Performance)
  2. Defer Third-Party Scripts: Load Yotpo, Clarity, Searchspring after user interaction (Performance)
  3. Reduce Unused JavaScript: 548-712KB savings available through code splitting (Performance)
  4. Enable Pinch-Zoom: Remove user-scalable=no from viewport meta (Accessibility)
  5. Fix Homepage CLS: Reserve space for dynamic content, set explicit image dimensions (Performance)

üö´ Key Accessibility Violations

  1. Pinch-Zoom Disabled (Critical):
    Viewport meta contains user-scalable=no and maximum-scale restrictions
    Impact: Users with low vision cannot zoom to read content
  2. ARIA Input Fields Without Names:
    Slick slider uses role="listbox" on <div class="slick-track"> without proper accessible names
    Impact: Screen reader users cannot understand slider content
  3. Heading Hierarchy Issues:
    H3 "Back In Stock" followed by H4 in footer ‚Äî non-sequential heading order
    Impact: Screen reader navigation is confusing
  4. Images Missing Alt Text:
    Preview images (likely video thumbnails) lack alt attributes
    Impact: Screen reader users miss content context
  5. Accessible Name Mismatches:
    Links with aria-label that don't match visible text (e.g., "GET REWARDS link")
    Impact: Voice control users may struggle to activate controls

‚ôø Accessibility Quick Wins

  1. Enable Pinch-Zoom (Critical):
    Change viewport meta from:
    <meta name="viewport" content="width=device-width, minimum-scale=1.0, maximum-scale=1.0">
    To: <meta name="viewport" content="width=device-width, initial-scale=1">
    Impact: Immediate compliance with WCAG 1.4.4
  2. Fix Slick Slider ARIA:
    Remove role="listbox" from slick-track or implement proper listbox pattern
    Alternative: Use simpler CSS-only announcements bar
    Impact: Screen reader users can navigate announcement content
  3. Correct Heading Order:
    Ensure headings follow H1 ‚Üí H2 ‚Üí H3 sequence without skips
    Use CSS for visual sizing rather than incorrect heading levels
    Impact: Improved screen reader navigation
  4. Add Alt Text to All Images:
    alt="{{ image.alt | default: product.title }}"
    Ensure video preview images have descriptive alt text
    Impact: Screen reader users understand all visual content
  5. Fix Accessible Name Mismatches:
    Ensure aria-label matches or includes visible text
    Example: aria-label="Get Rewards" (not "GET REWARDS link")
    Impact: Voice control and screen reader consistency

‚ö†Ô∏è Accessibility Compliance Note

The disabled pinch-zoom is a WCAG Level AA violation (Success Criterion 1.4.4 Resize Text). This affects users with low vision who rely on zoom functionality. Australian accessibility requirements increasingly align with WCAG 2.1 AA standards. Enabling zoom should be the immediate priority fix.
